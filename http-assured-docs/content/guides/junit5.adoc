= JUnit 5 Extension

The http-assured JUnit 5 extension manages client lifecycle and provides parameter injection. Add the dependency:

[source,xml,subs=attributes+]
----
<dependency>
    <groupId>io.smallrye</groupId>
    <artifactId>http-assured-junit5</artifactId>
    <version>{project-version}</version>
    <scope>test</scope>
</dependency>
----

=== Annotation Attributes

[cols="1,1,2"]
|===
|Attribute |Default |Description

|`baseUri` |`""` |Base URI for all requests
|`port` |`-1` |HTTP port (uses URI port if -1)
|`basePath` |`""` |Path prefix for all requests
|===

Apply fault injection to individual test methods:

[source,java]
----
include::{snippets-junit5}/JUnit5Snippets.java[tag=with-toxic-annotation]
----

The annotation wraps the client's HTTP engine in a `ToxicEngine` for that test method only. Other tests in the same class are unaffected.

=== Annotation Attributes

[cols="1,1,2"]
|===
|Attribute |Default |Description

|`latencyMs` |`0` |Delay in milliseconds
|`jitterMs` |`0` |Jitter range (used with latencyMs)
|`bandwidthBytesPerSecond` |`0` |Bandwidth limit
|`down` |`false` |Refuse connections
|`resetPeer` |`false` |Reset connection
|`resetPeerDelayMs` |`0` |Delay before reset
|`timeoutMs` |`0` |Timeout simulation
|`slowCloseMs` |`0` |Delay connection close
|`limitDataBytes` |`0` |Truncate after N bytes
|`respondWithStatus` |`0` |Return fixed status code
|`toxicity` |`1.0` |Probability (0.0 to 1.0)
|===
