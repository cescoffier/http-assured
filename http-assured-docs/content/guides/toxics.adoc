The `@WithToxic` annotation applies toxics declaratively on test methods. See the link:/guides/junit5[JUnit 5 Extension] guide.

== Available Toxics

[cols="1,1,2"]
|===
|Factory Method |Parameters |Description

|`Toxic.latency(delay)` |`Duration` |Adds fixed delay to requests
|`Toxic.latency(delay, jitter)` |`Duration, Duration` |Adds delay with randomized jitter
|`Toxic.down()` |none |Refuses the connection
|`Toxic.timeout(timeout)` |`Duration` |Accepts connection but never responds
|`Toxic.resetPeer()` |none |Resets connection immediately
|`Toxic.resetPeer(delay)` |`Duration` |Resets connection after delay
|`Toxic.bandwidth(bytesPerSec)` |`long` |Limits bandwidth
|`Toxic.slowClose(delay)` |`Duration` |Delays connection close
|`Toxic.limitData(bytes)` |`long` |Truncates response after byte limit
|`Toxic.respondWith(status)` |`int` |Returns fixed status code
|`Toxic.respondWith(status, body)` |`int, String` |Returns fixed status and body
|===

All toxics support `.withToxicity(double)` to set probability and `.upstream()` to apply on the request path instead of response path.
